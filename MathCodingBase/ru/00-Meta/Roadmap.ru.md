# Дорожная карта

Этот документ — рабочий план MathCoding. Его ведёт агент, который двигает методологию. Здесь перечислены пакеты задач, расширяющие методологию, в порядке приоритета, с текущим статусом. Статусы меняются по мере прохождения пакетов через `bin/verify` и `bin/validate-packet`.

## Условные обозначения статусов

- `pending` — работа не начата
- `in-progress` — пакет в работе
- `verified` — пакет проверен от начала до конца (`bin/verify` + `bin/validate-packet`)
- `blocked` — работа приостановлена до решения или внешнего ввода
- `cancelled` — работа не будет сделана; причина записана здесь же

## За пределами дорожной карты

Сюда не входят:

- Веб-интерфейс для методологии.
- Глубокая интеграция с плагинами IDE за пределами слоя адаптеров.
- Новые формализмы (Coq, Lean и др.) — отмечены как будущая работа в `TLA+ Role.md`, требуют экспертного знания каждого формализма.
- Выбор протокола на основе машинного обучения — спекулятивно, не приоритезировано.
- Поддержка API не-opencode агентов (Aider, Cody, Continue и др.) — только если реальный пользователь попросит перенос.

## Как обновлять этот документ

Когда пакет меняет состояние, агент правит соответствующую строку:

```markdown
- [ ] **1.1 ui-modal-react-runtime** — `pending`
```

превращается в

```markdown
- [x] **1.1 ui-modal-react-runtime** — `verified`
  Короткая заметка в одну строку о том, что именно сделано.
```

Отменённые пункты не удаляются. Им ставится статус `cancelled`, и причина остаётся здесь. Синтаксис чекбоксов читается человеком; исходник — markdown. Если будущему инструменту понадобится машино-читаемый статус, он разберёт токен `` — `статус` `` после заголовка.
