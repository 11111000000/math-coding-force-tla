# Модель переносимости агентов

Чтобы перенести MathCoding в другую агентскую среду, сохраните:

1. `schemas/`
2. документы протоколов из `MathCodingBase/02-Protocols/`
3. конвенцию каталогов пакетов задач
4. контракт механического верификатора, включая размеченное объединение `UNVERIFIABLE`

Затем адаптируйте слой интеграции:

- промпты
- команды
- права доступа к инструментам
- обёртки

Поэтому `.opencode/` — адаптер, а не ядро.

Перечисление вердиктов и подтипы `UNVERIFIABLE:*` принадлежат контракту верификатора. Перенос в новую среду означает сохранение объединения — `UNVERIFIABLE:TOOL_MISSING`, `UNVERIFIABLE:OUT_OF_SCOPE`, `UNVERIFIABLE:DEFERRED` — и обязательства `human_review`, которое идёт с каждым из них. Без этого неверифицируемость вырождается в отказ.
