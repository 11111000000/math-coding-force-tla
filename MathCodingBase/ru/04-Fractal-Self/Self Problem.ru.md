# Собственная задача

## Формулировка задачи

Разработать переносимую методологию, которая превращает агентскую разработку ПО из наивной генерации по правдоподобию в артефактоцентричную математическую разработку.

## Желаемый результат

Методология, которую любой достаточно способный кодирующий агент может принять, следуя:

- каноническим контрактам артефактов
- явным протоколам
- конвейеру «формальная модель раньше кода»
- верификации, несущей свидетельства
- честной неверифицируемости, где механическая проверка неприменима, — с именованными рецензентами и процессами

Методология должна подчиняться этим же правилам сама.
