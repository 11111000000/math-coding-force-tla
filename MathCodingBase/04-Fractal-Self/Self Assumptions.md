# Self Assumptions

1. Agent runtimes can read and write structured artifacts.
2. TLA+ is adequate as the default formal substrate for many tasks.
3. Mechanical verification is more trustworthy than prompt-only reasoning.
4. Developers will accept extra upfront rigor if downstream ambiguity is reduced.
5. Portability requires artifact-first rather than agent-first design.
6. Some artifact kinds (prose, UX, strategy, ethics) lie outside mechanical verification. The methodology names them rather than pretending they fit.
7. Human review with a named reviewer and a process is a real obligation when declared as compensation for unverifiability.
