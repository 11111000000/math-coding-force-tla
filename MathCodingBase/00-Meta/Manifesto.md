# Manifesto

MathCodingFractal exists to replace vibe-coding with math-coding.

## Thesis

Vibe-coding optimizes for local plausibility.
It yields code that looks right before we know what right means.

## Antithesis

Pure formalism without operationalization becomes ceremony.
If mathematics remains commentary after design decisions are already made, it is only decorative rigor.

## Synthesis

Math-coding means that mathematical artifacts become the development substrate.
The chat is not the spec. The code is not the first artifact. The verified model is not a luxury. It is the gate.

## Commitments

1. Every task becomes a task packet.
2. Every ambiguity becomes an explicit assumption.
3. Every non-trivial implementation must be preceded by refinement.
4. Every verification claim must have evidence.
5. Every methodological claim must be applicable to this repository itself.
