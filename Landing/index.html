Core Principles

What makes this different

◇

Artifact-Centered

Chat messages are transient. The packet is the source of truth. If it's not in a file, for the methodology it doesn't exist.

≡

Mechanical Verification

No "expert estimates." TLC exhaustively checks every reachable state. SANY rejects invalid specs. The tools are the gate.

⇌

Fractal Self-Application

The methodology describes itself through its own pipeline. `methodology/self-spec/` is verified TLA+ that models the methodology.

→

Refinement as Bridge

The jump from verified model to code is explicit. State mapping, operation mapping, invariant preservation — all documented.

✱

Explicit Assumptions

Every ambiguity becomes a tagged assumption: user-confirmed, agent-inferred, or open. No silent dependencies on open assumptions.

⧉

Traceability

Every model property maps to a code artifact. Every invariant has a preservation strategy. The map is machine-readable.

In Practice

From TLA+ to TypeScript

The same state machine, formalized and implemented.

Model.tla 
VARIABLES state, pendingResult

Init ==
  /\ state = "closed"
  /\ pendingResult = "none"

Open ==
  /\ state = "closed"
  /\ state' = "opening"
  /\ UNCHANGED pendingResult

Confirm ==
  /\ state = "open"
  /\ state' = "confirming"
  /\ pendingResult' = "in-flight"
dialogReducer.ts 
export function dialogReducer(
  state: DialogStateShape,
  action: DialogAction,
): DialogStateShape {
  switch (action.type) {
    case "OPEN":
      if (state.state !== "closed") return state;
      return {
        state: "opening",
        pendingResult: { kind: "none" }
      };
    // ... 10 actions total
  }
}
